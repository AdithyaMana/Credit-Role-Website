# Milestone: v2.0 (Builder Integration)

## Completed: 2026-03-03

## Overview
This milestone transformed the CRediT Taxonomy Visualizer from a read-only icon gallery into a full-featured **CRediT Builder** — an interactive tool scientists can use to build, visualize, and export CRediT author contribution matrices for their publications.

## Deliverables
- ✅ Codebase refactored with clean architecture and premium UI/UX
- ✅ Robust state management for contributor definitions (add/edit/remove/clear)
- ✅ 5 view modes: Matrix Grid, Standard Table, Heatmap, By Author, By Role
- ✅ Matrix Grid with click-to-assign and drag-to-assign role interactions
- ✅ Row/column swap for Matrix Grid view
- ✅ All 14 CRediT roles displayed simultaneously on desktop
- ✅ Academic monochrome styling matching scientific publication standards
- ✅ Export/Copy dropdown: JSON, CSV, Markdown, PNG
- ✅ Icons on Standard Table and Heatmap column headers
- ✅ Smart filtering — only assigned roles appear in non-edit views
- ✅ Mobile CRediT Builder with bottom tab bar navigation
- ✅ Mobile-optimized card-based editing with expandable role grids

## Key Technical Decisions
- **Separate desktop and mobile builder**: Desktop uses `BuilderView.tsx` with full-width layout; mobile uses `MobileBuilderView.tsx` with card-based editing optimized for touch
- **SVG heatmap abandoned in favor of HTML table**: SVG approach was cramped and inconsistent; HTML table matches Standard Table layout perfectly
- **Active role filtering**: All non-Matrix views compute `activeRoles` from contributor data, hiding unused columns
- **Academic monochrome aesthetic**: Consistent `border-slate-800` thick borders, serif fonts, and subtle icon headers matching APA-style formatting
- **Category-based icon coloring** in Matrix Grid using CRediT taxonomy categories (Strategy, Research, Infrastructure, Dissemination)

## Lessons Learned
- Scientists expect publication-ready output formats — CSV and Markdown are more useful than JSON for most workflows
- Heatmap views must exactly match sibling table layouts for visual consistency
- Mobile builder needs a fundamentally different interaction model (cards + expandable grids) vs the desktop matrix grid
- Bottom tab bar navigation is essential for mobile when the app has two distinct modes
